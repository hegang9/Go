# Go 语言映射表 (Map)

在 Go 语言中，`map` 是一种强大且内置的数据结构，用于存储键值对 (Key-Value pairs) 的无序集合。它类似于其他语言中的字典 (Dictionary)、哈希表 (Hash Table) 或关联数组。

## 1. 声明与初始化

### 1.1 仅声明 (nil map)
声明一个 map 需要指定键和值的类型。**注意：仅仅声明的 map 其值为 `nil`，不能直接用来存储键值对，必须先初始化。**
```go
// 声明一个键为 string，值为 int 的 map
var m map[string]int 
// 此时 m == nil，如果直接执行 m["age"] = 18 会引发 panic 错误
```

### 1.2 使用 make 函数初始化 (推荐)
使用内置的 `make` 函数可以分配内存并初始化 map。
```go
// 初始化一个空的 map
m1 := make(map[string]int)

// 也可以指定初始容量（提高性能，避免频繁扩容）
m2 := make(map[string]int, 10) 
```

### 1.3 使用字面量初始化
在声明的同时直接赋初值。
```go
m3 := map[string]string{
    "name": "Tom",
    "city": "Beijing",
}
```

## 2. 基本操作：增、删、改、查

### 2.1 添加与修改
向 map 中添加或修改数据非常简单，直接使用 `map[key] = value` 即可。如果 key 不存在则是添加，已存在则是修改。
```go
m := make(map[string]int)
m["Apple"] = 5  // 添加新元素
m["Apple"] = 10 // 修改已知元素的值
```

### 2.2 查找数据与 "Comma ok" 惯用法（深入解读）

在 Go 的 map 中进行键值查找时，有两种常见的表达式形式：

#### 形式 1：单返回值查找
```go
v := m[key]
```
如果键不存在，Go **不会报错**（不会像 Python 那样抛出 `KeyError` 导致程序崩溃），而是默默地**返回该值类型的零值**（比如 int 类型的零值是 `0`，string 的零值是 `""`，bool 的零值是 `false`）。
**这带来了一个“零值歧义”的问题：**
假设有一份成绩单 `m := map[string]int{"Tom": 85, "Jerry": 0}`。
- `m["Jerry"]` 查出来是 `0`（因为他确实考了0分）。
- `m["Bob"]` 查出来也是 `0`（因为 Bob 不存在，返回了 int 的零值）。
此时，光靠返回的 `0`，我们无法区分是谁参加了考试考 0 分，谁是缺考/查无此人。

#### 形式 2：双返回值查找 ("Comma ok" 惯用法)
为了解决上述歧义，Go 设计了带有两个返回值的查找语法：
```go
value, ok := m[key]
```
- `value`：返回键对应的值（存在时为其值，不存在时依然是零值）。
- `ok`：返回一个 **布尔值 (bool)**。如果查到了这个键，`ok` 为 `true`；如果没查到，`ok` 为 `false`。

**完整代码示例演示：**
```go
m := map[string]int{"Tom": 85, "Jerry": 0}

// 1. 存在且有正常值的键
score1, ok1 := m["Tom"]
// 输出: score1: 85, ok1: true

// 2. 存在，但值恰好是零值的键
score2, ok2 := m["Jerry"]
// 输出: score2: 0, ok2: true (因为 ok=true，说明 Jerry 确实参加了考试)

// 3. 完全不存在的键
score3, ok3 := m["Bob"]
// 输出: score3: 0, ok3: false (ok=false 说明 Bob 根本没录入系统)
```

#### 引申写法与技巧：

**1. 结合 `if` 语句的最常用紧凑写法**
Go 语言允许在 `if` 判断条件中加入一个初始化语句，这在获取 map 值时极其常见：
```go
if score, ok := m["Bob"]; ok {
    fmt.Println("Bob 的分数是:", score)
} else {
    fmt.Println("Bob 不存在于 map 中")
}
```

**2. 仅判断键是否存在，忽略值**
如果你只关心“名字在这份 map 里面吗？”，并不需要拿他的成绩，可以使用**空白标识符 `_`** 将值忽略掉，节省内存：
```go
if _, ok := m["Tom"]; ok {
    fmt.Println("Tom 在名单中")
}
```

### 2.3 删除元素
使用内置的 `delete` 函数可以从 map 中删除指定的键值对。如果该键本身就不存在，`delete` 什么也不做，也不会报错。
```go
m := map[string]string{"A": "Apple", "B": "Banana"}
delete(m, "A") // 删除键 "A"
```

## 3. 遍历 Map

和切片类似，使用 `for...range` 循环可以轻松遍历 map：
```go
m := map[string]int{"Math": 90, "English": 80}

for key, value := range m {
    fmt.Printf("键: %s, 值: %d\n", key, value)
}
```
**重要提示：** Map 的遍历是**无序的**。每次遍历 map 时的元素顺序可能完全不同，千万不要依赖 map 的遍历顺序编写业务逻辑。

## 4. Map 的核心特性与注意事项

1. **键(Key)的限制**：Map 的键必须是支持 `==` 和 `!=` 比较操作的类型，例如基本类型（int, string, float）、指针、通道(channel)、接口(interface)等。**切片(slice)、映射(map)、和包含切片的结构体(struct) 不能作为 map 的键。**
2. **引用类型**：和切片一样，`map` 是引用类型。将 map 传递给函数或赋值给另一个变量时，传递的是底层的指针。修改新变量会影响原 map。
3. **并发不安全**：Go 语言内置的 `map` **不是并发安全的**。如果在多个 goroutine 中同时对同一个 map 进行读写，会引发致命错误 (`fatal error: concurrent map read and map write`) 导致程序崩溃。在并发场景中，需要搭配 `sync.Mutex` 互斥锁，或者使用标准库提供的 `sync.Map`。
