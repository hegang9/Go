## 包
在 Go 中，程序是通过将包链接在一起来构建的。Go 中进行导入的**最基本单位是一个包**，而不是.go文件。包其实就是一个文件夹，英文名 package，包内共享所有变量，常量，以及所有定义的类型。包的命名风格建议都是**小写字母**，并且要尽量简短。

## 可见性
对于包内文件，共享所有变量，常量，以及所有定义的类型。对于包外文件，只有**首字母大写**的变量、常量以及类型才是可见的。Go 语言没有 public，private 这样的访问修饰符，只有**首字母大小写**来区分可见性。

## 导包语法
```go
package main

import (
  e "example"   // e 为别名，可以降低包名复杂度，也可以解决包名重复冲突问题
  e1 "example1"
  _ "example2"  // 匿名导入，该包无法被使用，仅用于执行包的 init() 函数，常用于注册驱动，但不需要手动使用驱动
)
```

导入后，通过`包名.标识符`去访问，如果导入时使用了别名，则通过`别名.标识符`去访问。

以下特殊的导入方法将该包中所有类型都导入到当前包作用域，可以直接访问包中的类型，而不需要通过`包名.标识符`的方式去访问：

```go
import . "example"
```

## 内部包
Go 中约定，一个包内名为 `internal` 的包为内部包，外部包将无法访问内部包中的任何内容，否则的话编译不通过。内部包只能被同一模块内的其他包导入，不能被外部模块导入。这种设计有助于保护包的内部实现细节，防止外部代码依赖于不稳定的接口。



## 注意事项
Go 中无法进行循环导入，无论直接还是间接循环，都会导致编译错误。
