# Go 语言数组与切片 (Array & Slice)

在 Go 语言中，数组（Array）和切片（Slice）是用于存储同类型元素集合的核心数据结构。了解它们之间的区别和联系是掌握 Go 语言的关键。

## 1. 数组 (Array)

数组是具有**固定长度**且拥有**相同数据类型**的元素序列。

### 1.1 数组的特点
- **长度固定**：数组在声明时必须指定长度，且长度一旦定义就不能更改。
- **值类型**：在 Go 语言中，数组是**值类型**（Value Type）。这意味着当把一个数组赋值给另一个数组，或者作为参数传递给函数时，会对整个数组进行一次**拷贝**。
- **类型组成**：数组的长度是其类型的一部分。因此 `[3]int` 和 `[4]int` 是两种完全不同的类型，不能相互比较或赋值。

### 1.2 声明与初始化
```go
// 1. 仅声明，默认初始化为对应类型的零值（如 int 为 0）
var a [5]int 
// a 的值为: [0 0 0 0 0]

// 2. 声明并初始化
var b = [3]int{1, 2, 3}

// 3. 简短声明
c := [4]int{1, 2, 3, 4}

// 4. 让编译器自动推导数组长度（使用 ...）
d := [...]int{10, 20, 30} 
// d 的类型被推导为 [3]int

// 5. 根据索引初始化特定元素（未指定的为 0）
e := [5]int{1: 10, 3: 30} 
// e 的值为: [0 10 0 30 0]

// 6. 通过new函数创建数组，返回一个指向数组的指针
f := new([5]int)
// f 的类型是 *[5]int，f 指向一个长度为 5 的数组，初始值为 [0 0 0 0 0]
```

## 2. 切片 (Slice)

因为数组长度固定且作为参数传递时拷贝开销大，Go 语言提供了一种更灵活、更强大的数据结构：**切片（Slice）**。你可以将切片理解为“动态数组”。

### 2.1 切片的底层结构
切片本身并不存储任何数据，它只是对底层数组的一个**引用（视图）**。一个切片在底层包含三个字段：
- **指针 (Pointer)**：指向底层数组中切片指定的开始位置。
- **长度 (Length)**：切片当前包含的元素个数。
- **容量 (Capacity)**：从切片的开始位置到底层数组末尾的元素个数。

### 2.2 切片的创建与初始化
```go
// 1. 从切片字面量创建（最常用）
s1 := []int{1, 2, 3} // 注意括号里没有数字也没有 ...，这就是切片

// 2. 从已有数组或切片中生成切片 
// 语法: slice[start:end] (包含 start, 不包含 end)
arr := [5]int{10, 20, 30, 40, 50}
s2 := arr[1:4] // 取出 arr 索引 1、2、3 的元素，所以 s2 = [20, 30, 40]

// 省略 start 默认从 0 开始: arr[:3] 等价于 arr[0:3]
// 省略 end 默认到数组末尾: arr[2:] 等价于 arr[2:len(arr)]
// s := arr[:] 则是获取一个包含数组全部元素的切片

// 3. 使用 make 函数提前分配内存 (推荐用于明确知道长度/容量的场景)
// make([]Type, length, capacity)
s3 := make([]int, 3, 5) // 创建一个长度为 3，容量为 5 的切片，初始值为 [0 0 0]
s4 := make([]int, 3)    // 容量参数可以省略，此时 容量 = 长度
```

### 2.3 `len()` 和 `cap()`
内置函数 `len()` 用于获取切片的长度，`cap()` 用于获取切片的容量。
```go
s := make([]int, 2, 5)
fmt.Printf("len=%d cap=%d slice=%v\n", len(s), cap(s), s)
// 输出: len=2 cap=5 slice=[0 0]
```

### 2.4 append() 添加元素
使用内置的 `append()` 函数可以动态向切片追加元素。如果底层数组容量不够，Go 会自动分配一个新的、更大的底层数组，并将原有数据复制过去（这被称为“扩容”）。
```go
var s []int // nil 切片
s = append(s, 1) // 追加一个元素
s = append(s, 2, 3) // 追加多个元素

s2 := []int{4, 5}
s = append(s, s2...) // 追加另一个切片的所有元素 (注意后面的 ... 不能省略)
```
*注意：`append` 操作可能会创建新的底层数组，因此必须将 `append` 的返回值重新赋值给原切片。*

### 2.5 切片的引用特性
因为切片是对引用的包装，所以修改切片的值，会直接修改底层数组对应的值；同时，共享同一个底层数组的其他切片也会观察到这个修改。
```go
arr := [3]int{1, 2, 3}
s := arr[:]
s[0] = 99
fmt.Println(arr) // 输出: [99 2 3]，底层数组被修改了！
```

### 2.6 copy() 函数
为了避免因为两个切片共享底层数组带来的意外修改，或者在只需要很小一部分数据时释放原本庞大的底层数组，可以使用内置函数 `copy()` 来深拷贝切片里的元素。
```go
src := []int{1, 2, 3}
dst := make([]int, len(src)) // 必须预先分配好目标切片的所需空间
copy(dst, src)
```

## 3. 数组总结与对比

| 特性 | 数组 (Array) | 切片 (Slice) |
| :--- | :--- | :--- |
| **长度** | 固定长度，声明后不可修改 | 动态长度，可通过 `append` 自动扩容 |
| **类型** | 值类型 (Value Type)，赋值会产生拷贝 | 引用类型 (Reference Type)，底层是指针包装 |
| **函数传参** | 复制整个数组，开销较大（一般传数组指针解决） | 复制切片头部信息（指针、长度、容量），开销极小 |
| **使用场景** | 确定元素数量且不再变化的场景，如加密算法中的哈希值存储 | 几乎所有的绝大多数业务代码场景 |
