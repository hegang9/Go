# Go 语言条件控制

在 Go 语言中，条件控制语句主要用于根据不同的条件执行不同的代码分支。主要的条件控制语句包括 `if`、`switch` 和 `select`。

## 1. if 语句

Go 语言的 `if` 语句与其他主流语言类似，但有两点显著的不同：
- 条件表达式不需要使用括号 `()` 括起来。
- 大括号 `{}` 是必须的，即使代码块只有一行。

### 基本语法
```go
if condition {
    // 当 condition 为 true 时执行的代码
}
```

### 包含初始化的 if
在 `if` 语句中，可以在条件表达式执行前添加一个执行语句，主要用于初始化变量。该变量的作用域仅在 `if` 代码块（包含 `else` 块）内部可见。
```go
if x := 10; x > 5 {
    fmt.Println("x 大于 5")
}
// 离开 if 语句块后，x 将无法被访问
```

## 2. if...else 和 if...else if

### if...else
```go
if condition {
    // 条件为真时执行
} else {
    // 条件为假时执行
}
```
**注意：** Go 语言有严格的格式要求，`else` 必须和 `if` 代码块的右大括号 `}` 在同一行，不能换行。

### if...else if...else
用于多个条件的判断：
```go
if condition1 {
    // condition1 为真
} else if condition2 {
    // condition1 为假，且 condition2 为真
} else {
    // 上述条件均为假
}
```

## 3. switch 语句

`switch` 语句用于基于不同的条件执行不同的动作。Go 语言的 `switch` 非常灵活，默认情况下每个 `case` 执行完毕后会自动退出，**不需要**像 C/C++ 那样在末尾写 `break`。

### 基本语法
```go
v := 2
switch v {
case 1:
    fmt.Println("数字是 1")
case 2:
    fmt.Println("数字是 2")
default:
    fmt.Println("未知数字")
}
```

### switch 的核心特性
1. **多条件匹配**：一个 `case` 可以携带多个表达式，用逗号分隔。
   ```go
   switch day {
   case "Monday", "Tuesday", "Wednesday", "Thursday", "Friday":
       fmt.Println("工作日")
   case "Saturday", "Sunday":
       fmt.Println("周末")
   }
   ```
2. **无条件 switch (代替 if-else 链)**：可以省略 `switch` 后的表达式，相当于 `switch true`，在此模式下，每个 `case` 后面跟着一个可以计算为布尔值的表达式。
   ```go
   score := 85
   switch {
   case score >= 90:
       fmt.Println("优秀")
   case score >= 80:
       fmt.Println("良好")
   case score >= 60:
       fmt.Println("及格")
   default:
       fmt.Println("不及格")
   }
   ```
3. **fallthrough 穿透**：如果需要强制执行紧随其后的下一个 `case` 的代码，可以使用 `fallthrough` 关键字。
   ```go
   switch num {
   case 1:
       fmt.Println("第一")
       fallthrough // 会继续无条件执行 case 2 的代码块
   case 2:
       fmt.Println("第二")
   }
   ```

## 4. select 语句

`select` 是 Go 提供的一种特殊的控制结构，类似于用于通信的 `switch` 语句。它的每个 `case` 都必须是一个通道（Channel）的发送或接收操作。它主要用于**并发编程**。

```go
select {
case msg1 := <-c1:
    fmt.Println("从 c1 接收到:", msg1)
case msg2 := <-c2:
    fmt.Println("从 c2 接收到:", msg2)
default:
    fmt.Println("没有接收到任何消息") // 如果 c1 和 c2 都没有准备好，就执行 default
}
```
`select` 会阻塞，直到其中的某个通信操作可以继续执行为止；如果多个 `case` 同时准备好，则会随机选择一个执行。
