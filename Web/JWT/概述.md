## JWT 概述
J:JSON

W:Web

T:Token

JWT（JSON Web Token）是一种基于 JSON 的开放标准（RFC 7519），用于在网络应用环境中安全地传输声明信息。JWT 由三部分组成：头部（Header）、载荷（Payload）和签名（Signature）。它们通过点（.）分隔开来。每一部分都是一个base64的编码字符串。

## Header
实际上是一个JSON，里面包含两项信息：
- `alg`：表示签名算法（加密算法），如 HMAC SHA256 或 RSA。
- `typ`：表示令牌类型，通常是 "JWT"。

## Payload
也是JSON，包含一些通用信息以及一些自定义属性，例如：
- `iss`：发行者（Issuer）
- `sub`：主题（Subject）
- `aud`：受众（Audience）
- `exp`：过期时间（Expiration Time）
- `nbf`：不早于（Not Before）
- `iat`：签发时间（Issued At）
- `jti`：JWT ID（JWT Identifier）

`PayLoad`中实际包含了用户的身份信息，一般只存储少量（为了性能）的非敏感信息（为了安全）。

## Signature
对前两部分进行哈希签名，具体来说，将前两部分通过 base64 编码后用点`.`连接起来，作为哈希签名算法的明文输入，再使用一个密钥经过 Header 中指定的算法进行处理，再将三者通过`.`连接形成最终签名（最初是一串二进制字节数组，后续可能会被进一步编码为base64字符串）。

## JWT 的优势
1. 无状态：JWT 是自包含的，服务器不需要存储会话信息，所有信息只在客户端存储，适合分布式系统。
2. 跨域支持：JWT 可以通过 HTTP 头部传输，适合跨域认证。

## JWT 的劣势
1. 安全性：JWT 是基于文本的，`Payload`没有被加密，容易被窃取和篡改，必须使用 HTTPS 来保护传输安全。
2. 不可撤销：一旦 JWT 被签发，就无法撤销，除非设置较短的过期时间或使用黑名单机制。

## JWT与普通令牌的区别
普通令牌（Opaque Token）通常是一个随机字符串，服务器需要在数据库中存储这个令牌以及相关的用户信息，每次验证时都需要查询数据库。而 JWT 是自包含的，所有信息都在令牌本身，服务器只需要验证签名即可，无需查询数据库。因此，JWT 更适合分布式系统和微服务架构，但需要注意安全性问题。
