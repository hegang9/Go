## JWT 工作流程

### 1. 用户登录
用户通过登录界面输入用户名和密码，提交到服务器进行验证。

### 2. 服务端验证用户信息
服务器接收到登录请求后，验证用户的身份信息（如用户名和密码）。如果验证通过，就进入第3步

### 3. 生成 JWT
服务端生成`Header`和`Payload`，并使用指定的算法和密钥生成`Signature`，最终将三部分通过点`.`连接成一个完整的 JWT。接着将JWT返回给客户端。

### 4. 客户端存储 JWT
客户端通常会将 JWT 存储在浏览器的 `localStorage` 或 `sessionStorage` 中，或者作为 HTTP-only cookie 存储，以便在后续请求中使用。本质上就是将JWT写进一个文件中。

### 5. 客户端发送请求
在后续的请求中，客户端会将 JWT 作为认证凭证发送给服务器，通常是通过 HTTP 请求头中的 `Authorization` 字段，格式为 `Bearer <token>`。

### 6. 服务端验证JWT
服务器接收到带有 JWT 的请求后，会验证 JWT 的签名是否有效，以及是否在有效期内。如果验证通过，就允许访问受保护的资源。


## JWT 验证
服务端拿到的JWT就是一个字符串，通过`.`分割成`Header`、`Payload`和`Signature`三部分。

1. 服务端首先取出前两部分，并使用相同的算法和**自己的密钥**对它们进行哈希签名，生成一个新的签名。
2. 然后将新生成的签名与 JWT 中的签名进行比较，如果两者匹配，说明 JWT 是有效的，用户可以访问受保护的资源；如果不匹配，则说明 JWT 无效，拒绝访问。
