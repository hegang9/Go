# Go语言泛型 (Generics)

Go 语言在 1.18 版本中正式引入了泛型（Generics）。泛型允许我们在编写函数、数据结构时，使用**类型参数**（Type Parameters）来代替具体的类型，从而实现代码的复用，减少重复的样板代码。

## 1. 为什么需要泛型？

在泛型引入之前，如果我们想写一个比较两个数大小并返回较大值的函数，我们需要为每种类型写一个：

```go
func MaxInt(a, b int) int { ... }
func MaxFloat64(a, b float64) float64 { ... }
```

有了泛型之后，我们可以使用**类型参数**将类型抽象出来，只写一次核心逻辑。

## 2. 泛型函数与类型参数

泛型函数是在常规函数名称之后、参数列表之前，增加一个用中括号 `[]` 括起来的**类型参数列表**。

```go
// T 是类型参数，any 是对 T 的类型约束（表示可以是任何类型）
func PrintSlice[T any](s []T) {
    for _, v := range s {
        fmt.Println(v)
    }
}

func main() {
    PrintSlice[int]([]int{1, 2, 3})
    PrintSlice[string]([]string{"Hello", "World"})
    
    // 借助类型推断，通常可以省略显式的类型实参 [T]
    PrintSlice([]float64{1.1, 2.2})
}
```

## 3. 类型约束 (Type Constraints)

类型参数不能是毫无限制的，如果要在函数里使用 `+`、`>` 等操作符，必须告诉编译器这个类型支持这些操作。这就是**类型约束**。

Go 语言使用**接口**（Interface）来作为类型约束。

### 3.1 内置的约束

*   `any`：就是 `interface{}`，表示没有任何限制，支持任何类型。
*   `comparable`：内置的约束，表示类型支持通过 `==` 和 `!=` 进行比较操作（例如 int, string, bool 等，但不支持 slice, map）。

```go
// T 必须实现了 comparable，否则无法在 slice 中查找或者使用 == 进行比较
func Index[T comparable](s []T, x T) int {
    for i, v := range s {
        if v == x {
            return i
        }
    }
    return -1
}
```

### 3.2 自定义类型约束（接口的新用法）

在泛型中，接口除了定义方法集，还可以用来定义**类型集**（Type Sets）。我们可以在接口中使用 `|`（并集）来指定允许的具体类型。

```go
// 定义一个类型约束，只允许 int, float64 以及它们的底层类型
type Number interface {
    int | int8 | int16 | int32 | int64 | float32 | float64
}

// 现在的 T 被限制为只能是 Number 接口里列出的那些类型
func SumPointers[T Number](m map[string]T) T {
    var s T
    for _, v := range m {
        s += v // 因为约束里都是数值类型，所以编译器允许使用 + 运算符
    }
    return s
}
```

### 3.3 `~` 符号（底层类型约束）

如果我们使用 `type MyInt int` 定义了一个新类型，`MyInt` 虽然底层是 `int`，但它不属于 `int` 类型。
为了让 `MyInt` 也能匹配 `int` 的约束，可以使用 `~` 符号指示**底层类型**必须匹配。

```go
type Ordered interface {
    ~int | ~int8 | ~int16 | ~int32 | ~int64 |
    ~uint | ~uint8 | ~uint16 | ~uint32 | ~uint64 | ~uintptr |
    ~float32 | ~float64 |
    ~string
}

// 这样自己定义的 MyInt 也能传入这个泛型函数了
type MyInt int

func Min[T Ordered](a, b T) T {
    if a < b {
        return a
    }
    return b
}
```

## 4. 泛型类型 (Generic Types)

除了函数，我们还可以定义基于类型参数的数据结构，也就是泛型类型。典型应用是泛型切片、泛型映射、泛型结构体等。

### 4.1 泛型结构体

```go
// T 可以是任何类型
type Stack[T any] struct {
    items []T
}

func main() {
    // 实例化为存放 int 的栈
    var intStack Stack[int]
    intStack.items = append(intStack.items, 1)

    // 实例化为存放 string 的栈
    var stringStack Stack[string]
    stringStack.items = append(stringStack.items, "Go")
}
```

### 4.2 泛型方法

可以为泛型类型定义方法（注意，泛型方法的接收者必须也是泛型的，且带上类型参数）。

**注意：Go 的结构体方法本身不支持独立的类型参数。**
这与 C++ 或 Java 等语言不同。在 Go 中，你**不能**为某个结构体的方法单独去定义一个新的泛型变量，方法里能使用的泛型，必须是由它的结构体（接收者）提供的。

**错误示例（Go 不允许的做法）：**
```go
type Processor struct{}

// ❌ 编译报错：方法无法拥有自己的独立泛型参数 [U any]
func (p *Processor) DoSomething[U any](val U) {
    fmt.Println(val)
}
```

**正确做法：**
如果必须要实现类似的功能，不能用方法，只能写成**普通的泛型函数**：
```go
// ✔️ 普通的泛型函数是可以包含独立类型参数的
func DoSomething[U any](p *Processor, val U) {
    fmt.Println(val)
}
```

**标准的泛型方法语法：**（方法的泛型 `T` 必须从 Receiver 的 `[T]` 继承过来）
```go
// 给前面的 Stack 类型添加方法
func (s *Stack[T]) Push(item T) {
    s.items = append(s.items, item)
}

func (s *Stack[T]) Pop() (T, bool) {
    if len(s.items) == 0 {
        var zero T // 声明类型 T 的零值
        return zero, false
    }
    top := s.items[len(s.items)-1]
    s.items = s.items[:len(s.items)-1]
    return top, true
}
```

## 5. 泛型中的类型推断

多数情况下，调用泛型函数时不需要显式地指定类型实参（如 `PrintSlice[int](...)`）。Go 编译器能够根据传入的常规参数值，自动推断出 `T` 的具体类型。

```go
func Multiply[T int | float64](a, b T) T {
    return a * b
}

func main() {
    // 编译器看到传入的是 5 和 6，推断出 T = int
    res1 := Multiply(5, 6) 
    
    // 编译器推断 T = float64
    res2 := Multiply(2.5, 3.0) 
}
```
