# Go语言文件与目录处理

在 Go 语言中，文件处理的核心接口位于 `os`、`io` 和 `bufio` 这三个标准库包中。

> **特别注意：** Go 1.16 版本之后，原来常用的 `io/ioutil` 包的大部分功能（比如 ioutil.ReadFile）已经被直接移到了 `os` 和 `io` 包中。建议在现代 Go 项目中**直接使用 `os.ReadFile` 和 `os.WriteFile`**。

---

## 1. 快速读写整个文件（适合小文件）

如果你的文件不大（比如配置文件、证书等），最简单的方法是一次性将整个文件读入内存，或者一次性覆盖写入。

### 1.1 快速读取：`os.ReadFile`
返回一个字节切片 `[]byte`，你需要将其转换为 `string` 才能打印文本。

```go
package main

import (
    "fmt"
    "os"
)

func main() {
    // 读取整个文件内容
    content, err := os.ReadFile("config.json")
    if err != nil {
        fmt.Println("读取文件失败:", err)
        return
    }
    fmt.Println("文件内容:\n", string(content))
}
```

### 1.2 快速写入：`os.WriteFile`
如果文件不存在会**自动创建**（根据你传入的权限，如 `0644`）。如果文件已存在，会**直接清空并覆盖**原来的内容。

```go
func main() {
    data := []byte("这是我们要写入的字符串内容\nHello Go!")
    
    // 参数：文件名, 字节内容, 文件权限(一般普通文件给 0644 即可)
    err := os.WriteFile("output.txt", data, 0644)
    if err != nil {
        fmt.Println("写入失败:", err)
        return
    }
    fmt.Println("写入成功！")
}
```

## 2. 基础的文件打开与关闭（资源管理）

如果文件很大，不能一次性放入内存，我们就必须“打开文件”得到一个文件句柄（指针），进行操作后，**必须关闭文件**释放系统资源。

```go
func main() {
    // 1. 只读模式打开文件
    file, err := os.Open("datalog.txt")
    if err != nil {
        fmt.Println("打开文件失败:", err)
        return
    }
    
    // 2. 及其重要：立刻 defer Close()！
    // 确保函数退出前，不管是否发生 panic，文件都会被妥善关闭
    defer file.Close()
    
    // 3. 执行你的文件读写操作...
    // file.Read(...)
}
```

*注：* `os.Open()` 是**只读模式**。如果你想修改文件，它会报错。\
对应的，`os.Create()` 是**只写模式**，且会自动清空同名旧文件。

## 3. 高级控制：追加内容 (`os.OpenFile`)

如果你不想覆盖旧文件，而是想**在文件末尾追加日志**，必须使用更底层的函数 `os.OpenFile`，它允许你指定具体的文件模式标志（Flag）。

```go
func main() {
    // O_APPEND: 追加模式
    // O_CREATE: 如果文件不存在就创建
    // O_WRONLY: 只写模式
    flags := os.O_APPEND | os.O_CREATE | os.O_WRONLY
    
    file, err := os.OpenFile("system.log", flags, 0644)
    if err != nil {
        fmt.Println("无法打开日志:", err)
        return
    }
    defer file.Close()
    
    // 写入字符串的内容
    _, err = file.WriteString("2024-01-01 12:00:00: User Login\n")
    if err != nil {
        fmt.Println("追加内容失败:", err)
    }
}
```

## 4. 带缓冲的高效读写与按行处理 (`bufio`)

在处理非常大的文件（例如几十 GB 的日志）时，推荐使用 `bufio` 包。它提供的缓冲区可以极大减少对操作系统的调用次数，提升读写速度。

### 4.1 大文件按行读取 (`bufio.Scanner`)

这可以说是读取大容量文本文件**最常用的标准姿势**：

```go
import (
    "bufio"
    "fmt"
    "os"
)

func main() {
    file, err := os.Open("huge_novel.txt")
    if err != nil { return }
    defer file.Close()

    // 创建一个基于文件句柄的 Scanner
    scanner := bufio.NewScanner(file)
    
    lineCount := 0
    // 每次调用 scanner.Scan() 就会读入新的一行，返回 true；读到文件末尾返回 false
    for scanner.Scan() {
        lineCount++
        // scanner.Text() 获取当前行的字符串内容（不包含换行符）
        lineText := scanner.Text()
        if lineCount <= 5 { // 这里我们只打印前5行作演示
            fmt.Printf("第 %d 行: %s\n", lineCount, lineText)
        }
    }

    // 检查扫描过程中是否发生了任何意外错误 (到达文件结尾EOF不算错误)
    if err := scanner.Err(); err != nil {
        fmt.Println("按行读取发生错误:", err)
    }
}
```

### 4.2 高效缓冲写入 (`bufio.Writer`)

当你需要密集地循环执行数万次的小段写入时，使用 `Writer` 攒够一批数据再提交给硬盘，性能能提升几十甚至上百倍。

```go
func main() {
    file, _ := os.Create("bulk_data.txt")
    defer file.Close()

    writer := bufio.NewWriter(file) // 默认缓冲区大小为 4096 字节
    
    for i := 0; i < 10000; i++ {
        // 先写到内存缓冲区里，并没有真的写到硬盘
        writer.WriteString(fmt.Sprintf("这是数据的第 %d 行\n", i))
    }
    
    // 重要！写完之后，必须把缓冲区里没满的数据“刷”进硬盘里
    writer.Flush() 
}
```

## 5. 目录/文件夹操作

Go 处理目录也非常简单直观：

```go
func main() {
    // 1. 创建单层目录 (如已存在会报错)
    os.Mkdir("myfolder", 0755)
    
    // 2. 创建多层嵌套目录 (像 linux 的 mkdir -p)
    os.MkdirAll("path/to/my/nested/folder", 0755)
    
    // 3. 获取目录下的文件列表
    entries, err := os.ReadDir(".") // 读取当前目录
    if err == nil {
        for _, entry := range entries {
            if entry.IsDir() {
                fmt.Println("文件夹:", entry.Name())
            } else {
                fmt.Println("文件:", entry.Name())
            }
        }
    }
    
    // 4. 删除单个空文件或空目录
    // os.Remove("myfolder") 
    
    // 5. 递归删除目录及其下所有文件 (危险操作，类似 rm -rf)
    // os.RemoveAll("path") 
}
```

### 小结与实战提示

1.  代码中但凡遇到 `os.Open` 或 `os.Create` 取到了句柄，下一行直接敲出 `defer file.Close()`，这是肌肉记忆。
2.  尽量弃用已被标记过期的 `io/ioutil` 库。
3.  不要轻易对 10MB 以上的文件使用 `os.ReadFile`，内存扛不住，一律老老实实上 `bufio.Scanner` 对行迭代，或者申请固定 `[]byte` 切片每次 `io.ReadFull` 块读取。
4.  如果你修改了已有的文件，一定要明确到底是截断清空重写（`os.Create`）还是往后追加（带有 `os.O_APPEND` 的 `os.OpenFile`）。
