# Go语言迭代器 (Iterators)

在 Go 1.23 版本中，Go 官方正式将**迭代器 (Iterators)** 标准化。这意味着 `for ... range` 循环不再局限于内置的 slice、map、channel 和字符串，而是可以**遍历自定义的函数**（这被称为 Range over func）。

为了配合这个新特性，Go 标准库引入了全新的 `iter` 包，并在 `slices` 和 `maps` 等包中添加了大量的迭代器辅助函数。

## 1. 什么是 Go 迭代器？

在 Go 中，迭代器本质上是一个**高阶函数**，它接收一个名为 `yield` 的回调函数作为参数。`for range` 循环在底层会自动生成这个 `yield` 函数并传递给你的迭代器。

标准库 `iter` 包主要定义了两种迭代器签名（代表单值和双值迭代，正好对应 `for v := range` 和 `for k, v := range`）：

```go
package iter

// 单值迭代器：每次 yield 产生一个值。
// 例如：按顺序生成一系列数字
type Seq[V any] func(yield func(V) bool)

// 双值迭代器：每次 yield 产生两个值（通常相当于索引/键 和 值）。
// 例如：遍历 map、遍历带索引的 slice
type Seq2[K, V any] func(yield func(K, V) bool)
```

**理解 `yield`：**
`yield` 是将值推送给 `for range` 循环主体的行为。如果 `for range` 内部执行了 `break`，`yield` 会返回 `false`，通知迭代器函数应当立即停止生成数据并返回。

## 2. 基础示例：自定义迭代器

我们来实现一个简单的单值迭代器，比如生成指定范围内的整数 `[start, end]`：

```go
package main

import (
    "fmt"
    "iter"
)

// RangeNum 返回一个单值迭代器 iter.Seq[int]
func RangeNum(start, end int) iter.Seq[int] {
    return func(yield func(int) bool) {
        for i := start; i <= end; i++ {
            // 将值推给循环体。如果 yield 返回 false (代表外部调用了 break)，则直接退出
            if !yield(i) {
                return
            }
        }
    }
}

func main() {
    // 像遍历切片一样遍历一个函数！
    for n := range RangeNum(1, 5) {
        fmt.Printf("%d ", n) // 输出: 1 2 3 4 5
    }
    fmt.Println()
    
    // 测试 break
    for n := range RangeNum(1, 10) {
        if n == 4 {
            break // 触发 yield 返回 false，迭代器内部终止
        }
        fmt.Printf("%d ", n) // 输出: 1 2 3
    }
}
```

## 3. 标准库中的迭代器

Go 1.23 为标准库增加了很多开箱即用的迭代器方法。例如在 `slices`（切片）和 `maps`（映射）包中。

### 3.1 `slices` 包

```go
package main

import (
    "fmt"
    "slices"
)

func main() {
    names := []string{"Alice", "Bob", "Charlie"}

    // 1. slices.All() 返回 Seq2[int, string]，类似常用的 range
    // 2. slices.Backward() 倒序遍历（非常实用！）
    for i, name := range slices.Backward(names) {
        fmt.Printf("Index: %d, Name: %s\n", i, name)
    }
    // 输出:
    // Index: 2, Name: Charlie
    // Index: 1, Name: Bob
    // Index: 0, Name: Alice
}
```

### 3.2 `maps` 包

`maps` 包提供了提取键或值的专门迭代器：

```go
package main

import (
    "fmt"
    "maps"
)

func main() {
    scores := map[string]int{"Alice": 90, "Bob": 80}

    // 只需要遍历 Keys (返回 iter.Seq 单值迭代器)
    for k := range maps.Keys(scores) {
        fmt.Println("Key:", k)
    }

    // 只需要遍历 Values
    for v := range maps.Values(scores) {
        fmt.Println("Value:", v)
    }
}
```

## 4. 复杂数据结构遍历 (以二叉树为例)

迭代器最强大的地方在于遍历复杂的数据结构，无需再手动压栈或返回包含所有元素的切片。你可以一边遍历一边产出数据。

```go
// 简单的二叉查找树节点
type Node struct {
    Val   int
    Left  *Node
    Right *Node
}

// InOrder 返回一个中序遍历的迭代器
func (n *Node) InOrder() iter.Seq[int] {
    return func(yield func(int) bool) {
        if n == nil {
            return
        }
        // 遍历左子树 (如果不继续，直接短路)
        n.Left.InOrder()(func(v int) bool {
            return yield(v)
        })
        
        // 生成当前节点
        if !yield(n.Val) {
            return
        }

        // 遍历右子树
        n.Right.InOrder()(func(v int) bool {
            return yield(v)
        })
    }
}
```

## 5. Pull (拉取式) 迭代器

前面提到的都是基于 `for range` 的 **“推式 (Push) 迭代器”**：迭代器主动执行并将值推送(yield)出来。
有时候，我们需要手动拿取“下一个”值（例如并行比较两个迭代器等场景）。标准库提供了 `iter.Pull` 将“推式”转换为“拉取式 (Pull)”。

```go
import "iter"

func main() {
    seq := RangeNum(1, 10)
    
    // 使用 iter.Pull 获取 next() 函数和 stop() 函数
    next, stop := iter.Pull(seq)
    defer stop() // 确保释放资源
    
    // 手动调用 next()，它会返回 (值, 是否有有效值)
    val1, ok1 := next() 
    fmt.Println(val1, ok1) // 1 true

    val2, ok2 := next()
    fmt.Println(val2, ok2) // 2 true
}
```

## 6. 总结

1. Go 1.23 引入的迭代器极大增强了语言的表达能力，它的原理是**基于函数的 `for range`**。
2. 核心是通过 `iter.Seq` 和 `iter.Seq2` 以及 `yield` 回调函数，由生成方控制数据流，消费方 (`for range`) 决定是否继续接收。
3. 它在处理惰性计算、无限序列生成、极其复杂的内存结构遍历时能发挥巨大的作用（省掉了组装切片的内存）。
