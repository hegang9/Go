# Go语言迭代器 (Iterators)

在 Go 1.23 版本中，Go 官方正式将**迭代器 (Iterators)** 标准化。这意味着 `for ... range` 循环不再局限于内置的 slice、map、channel 和字符串，而是可以**遍历自定义的函数**（这被称为 Range over func）。

为了配合这个新特性，Go 标准库引入了全新的 `iter` 包，并在 `slices` 和 `maps` 等包中添加了大量的迭代器辅助函数。

## 1. 什么是 Go 迭代器？

在很多编程语言 (如 Python) 中，迭代器自带状态，通常通过调用类似 `next()` 的方法拉取下一个值。但 Go 1.23 的迭代器走的是完全不同的路线：**基于高阶函数的“推送式”设计**。

简单来说，Go 中所谓的迭代器，**本身只是一个普通的函数**。这个函数接收另一个函数作为参数，我们把接收的这个参数函数约定俗成地叫作 `yield`。

标准库 `iter` 包定义了两种最基本的迭代器函数签名：

```go
package iter

// 单值迭代器：每次产出一个值 V
type Seq[V any] func(yield func(V) bool)

// 双值迭代器：每次产出两个值 K 和 V（类似 map 的键值对，或 slice 的索引和元素）
type Seq2[K, V any] func(yield func(K, V) bool)
```

你可以这样通俗地理解双方的角色：
- **迭代器（即 Seq 函数）**：是数据的**生产者**。它内部包含循环或逻辑，负责寻址或生成数据。
- **yield 函数**：是数据的**消费者**。生产者每拿到一个数据，就会调用一次 `yield`，把数据“塞”给消费者。
- **yield 的返回值 (bool)**：消费者如果返回 `true`，意思是“我还要下一个，继续发”；如果返回 `false`，意思是“够了，触发了 break，请停下吧”。

### 1.2 迭代器的核心实现原理：控制权反转与编译器重写

要真正看懂 Go 的迭代器机制，关键在于搞清楚**当我们写下 `for range` 循环时，编译器到底在背后做了什么手脚**。

Go 迭代器并没有借助任何复杂的“协程 (Goroutine)”或者“上下文保护”黑科技。它的本质是**编译器层面的语法糖 —— 编译器把你的循环体变成了一个闭包函数**。

让我们看一个简单直观的伪代码拆解：

**第一步：写一个“迭代器函数”**

什么是迭代器函数？**只要你的函数长成 `func(yield func(T) bool)` 这个样子，它就是迭代器函数。**
你可以把它当成一个“数据发射器”。我们写一个名为 `seq` 的迭代器函数，它会按顺序发射数字 1 和 2：

```go
// 这是一个标准的“迭代器函数”
func seq(yield func(int) bool) {
    yield(1) // 把 1 发送给消费者
    yield(2) // 把 2 发送给消费者
}
```

**第二步：你写的正常调用代码**

因为 `seq` 符合上面说的特殊签名，所以 Go 语言允许你直接把它放在 `for range` 的后面：

```go
// 直接遍历上面定义的 seq 函数
for v := range seq {
    fmt.Println("接收到:", v)
    if v == 1 {
        break // 收到 1 就喊停
    }
}
```

**第三步：编译器在底层悄悄转换成的代码**

编译器会把大括号 `{}` 里的**整个循环体**，直接打包成一个匿名的 `yield` 函数，然后把它作为参数传给你的 `seq` 迭代器执行。底层等价逻辑大致如下：

```go
// 1. 发起调用：把一个匿名函数传给迭代器 seq
seq(func(v int) bool {
    // 2. 你的 for 循环体内容被原封不动搬到了这里：
    fmt.Println("接收到:", v)
    
    // 3. 编译器对控制流语句(break/continue)做了翻译：
    if v == 1 {
        // 原本的 break，被翻译成 return false，告诉迭代器“停止推送”
        return false 
    }
    
    // 原本的循环走到底(或遇到 continue)，被翻译成 return true，告诉迭代器“继续推下一个”
    return true 
})
```

**我们像 CPU 一样，一步步走一遍上面的执行流程：**

1. **启动**：程序开始调用 `seq` 函数，并将上述打包好的匿名函数（消费者）当做 `yield` 参入传入。
2. **生产**：控制权在 `seq` 内部。`seq` 产生第一个值（假设是 `1`），并立刻调用 `yield(1)`。
3. **消费**：控制权**跳跃**到了你的 for 循环体里，`v` 被赋值为 `1`。执行打印后，遇到了判定条件。
4. **中断**：因为触发了 `v == 1`（即原代码的 `break`），闭包函数返回 `false`。此时，控制权**跳回**到 `seq` 函数内部。
5. **退出**：`seq` 发现刚才调用的 `yield(1)` 返回值居然是 `false`，它知道外部喊停了。于是 `seq` 内部放弃生产剩余数据，直接执行 `return`。遍历彻底结束。

**为什么要这样设计？**
这就是典型的“控制权反转”。遍历的节奏不是由消费者（`for` 循环）一遍遍去拔取的，而是由生产者（迭代器函数）主导、把数据挨个推送给消费者。
这种设计最大的好处是：**极致轻量级**。整个过程仅仅只是几次纯粹的“普通函数调用栈跳跃”，没有任何阻塞、并发或者调度开销，非常符合 Go 追求极简与高性能的设计哲学。

### 1.3 官方标准类型别名：iter.Seq 与 iter.Seq2

为了代码的可读性和统一标准，Go 官方在 `iter` 包中提供了两个非常重要的泛型类型别名，专门用来代替臃肿的嵌套函数签名：

**1. `iter.Seq[V]` (单值迭代)**
完全等价于 `func(yield func(V) bool)`。
* **使用场景**：只需要产出一个值的场景，例如遍历一组无索引的元素、生成一个数学序列、或者树的节点值遍历。
* **配合 `for range`**：直接对应单变量接收模式，即 `for v := range seq {}`。

**2. `iter.Seq2[K, V]` (双值迭代)**
完全等价于 `func(yield func(K, V) bool)`。
* **使用场景**：需要同时成对产出两个值的场景，最典型的是“键值对 (Key-Value)”或“索引+元素 (Index-Value)”。例如遍历 map，或类似于 `enumerate` 带着当前序号遍历数据。
* **配合 `for range`**：对应双变量接收模式，即 `for k, v := range seq2 {}`。

> **为什么要用别名？**
> 想象一下，如果不提供这组类型别名，我们要写一个“产出单值 int 迭代器的工厂函数”，签名会长这样：
> `func MakeIter() func(func(int) bool)` —— 嵌套的括号非常难看，容易让人眼花缭乱。
> 通过引入 `iter.Seq[int]`，上面的签名变成了极其清晰的：
> `func MakeIter() iter.Seq[int]`。
> 官方也鼓励开发者在公开 API 中尽量使用 `iter.Seq` 和 `iter.Seq2` 来传递迭代器，从而建立统一的规范。

## 2. 基础示例：自定义迭代器

我们来实现一个简单的单值迭代器，比如生成指定范围内的整数 `[start, end]`：

```go
package main

import (
    "fmt"
    "iter"
)

// RangeNum 返回一个单值迭代器 iter.Seq[int]， iter.Seq[int]是iter包里提供的一个官方类型别名（泛型），等价于 func(yield func(int) bool)
func RangeNum(start, end int) iter.Seq[int] {
    return func(yield func(int) bool) {
        for i := start; i <= end; i++ {
            // 将值推给循环体。如果 yield 返回 false (代表外部调用了 break)，则直接退出
            if !yield(i) {
                return
            }
        }
    }
}

func main() {
    // 像遍历切片一样遍历一个函数！
    for n := range RangeNum(1, 5) {
        fmt.Printf("%d ", n) // 输出: 1 2 3 4 5
    }
    fmt.Println()
    
    // 测试 break
    for n := range RangeNum(1, 10) {
        if n == 4 {
            break // 触发 yield 返回 false，迭代器内部终止
        }
        fmt.Printf("%d ", n) // 输出: 1 2 3
    }
}
```

## 3. 标准库中的迭代器

Go 1.23 为标准库增加了很多开箱即用的迭代器方法。例如在 `slices`（切片）和 `maps`（映射）包中。

### 3.1 `slices` 包

```go
package main

import (
    "fmt"
    "slices"
)

func main() {
    names := []string{"Alice", "Bob", "Charlie"}

    // 1. slices.All() 返回 Seq2[int, string]，类似常用的 range
    // 2. slices.Backward() 倒序遍历（非常实用！）
    for i, name := range slices.Backward(names) {
        fmt.Printf("Index: %d, Name: %s\n", i, name)
    }
    // 输出:
    // Index: 2, Name: Charlie
    // Index: 1, Name: Bob
    // Index: 0, Name: Alice
}
```

### 3.2 `maps` 包

`maps` 包提供了提取键或值的专门迭代器：

```go
package main

import (
    "fmt"
    "maps"
)

func main() {
    scores := map[string]int{"Alice": 90, "Bob": 80}

    // 1. 只需要遍历 Keys (返回 iter.Seq 单值迭代器)
    for k := range maps.Keys(scores) {
        fmt.Println("Key:", k)
    }

    // 2. 只需要遍历 Values
    for v := range maps.Values(scores) {
        fmt.Println("Value:", v)
    }
    
    // 3. 使用 maps.All() 同时提取键和值 (返回 iter.Seq2 双值迭代器)
    for k, v := range maps.All(scores) {
        fmt.Printf("Key: %s, Value: %d\n", k, v)
    }
}
```

## 4. 复杂数据结构遍历 (以二叉树为例)

迭代器最强大的地方在于遍历复杂的数据结构，无需再手动压栈或返回包含所有元素的切片。你可以一边遍历一边产出数据。

```go
// 简单的二叉查找树节点
type Node struct {
    Val   int
    Left  *Node
    Right *Node
}

// InOrder 返回一个中序遍历的迭代器
func (n *Node) InOrder() iter.Seq[int] {
    return func(yield func(int) bool) {
        if n == nil {
            return
        }
        // 遍历左子树 (如果不继续，直接短路)
        n.Left.InOrder()(func(v int) bool {
            return yield(v)
        })
        
        // 生成当前节点
        if !yield(n.Val) {
            return
        }

        // 遍历右子树
        n.Right.InOrder()(func(v int) bool {
            return yield(v)
        })
    }
}
```

## 5. Pull (拉取式) 迭代器

前面提到的都是基于 `for range` 的 **“推式 (Push) 迭代器”**：迭代器主动执行并将值推送(yield)出来。
有时候，我们需要手动拿取“下一个”值（例如并行比较两个迭代器等场景）。标准库提供了 `iter.Pull` 将“推式”转换为“拉取式 (Pull)”。

```go
import "iter"

func main() {
    seq := RangeNum(1, 10)
    
    // 使用 iter.Pull 获取 next() 函数和 stop() 函数
    next, stop := iter.Pull(seq)
    defer stop() // 确保释放资源
    
    // 手动调用 next()，它会返回 (值, 是否有有效值)
    val1, ok1 := next() 
    fmt.Println(val1, ok1) // 1 true

    val2, ok2 := next()
    fmt.Println(val2, ok2) // 2 true
}
```

## 6. 总结

1. Go 1.23 引入的迭代器极大增强了语言的表达能力，它的原理是**基于函数的 `for range`**。
2. 核心是通过 `iter.Seq` 和 `iter.Seq2` 以及 `yield` 回调函数，由生成方控制数据流，消费方 (`for range`) 决定是否继续接收。
3. 它在处理惰性计算、无限序列生成、极其复杂的内存结构遍历时能发挥巨大的作用（省掉了组装切片的内存）。
