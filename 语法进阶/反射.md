# Go语言反射 (Reflection) 深度解析

在静态强类型的 Go 语言中，所有的变量在编译时就确定了类型。但有时候我们需要在**运行时(Runtime)** 检查变量的类型、获取结构体的字段、甚至动态地修改变量的值或调用其方法，这时候就需要用到**反射 (Reflection)**。

Go 语言通过内置的 `reflect` 包提供反射功能。虽然反射极其强大（它是 `fmt.Println`、`json.Marshal` 等无数标准库和 ORM 框架背后的基石），但它的逻辑也相对晦涩。

---

## 1. 什么是反射？为什么要用它？

简单来说，反射是程序在**运行期间“审视”自己**的能力。它可以：
1. 检查一个未知的接口变量 (`any` / `interface{}`) 底层到底是什么类型。
2. 动态读取一个结构体里所有的字段名、字段类型和 Tag 标签（最常见的用法）。
3. 动态修改变量的值（即使你在写代码时不知道这个变量的具体类型）。
4. 动态调用变量身上绑定的方法。

**典型场景：** JSON 的序列化和反序列化 (`encoding/json`)、ORM 查询（如 `GORM`）、依赖注入驱动等。

---

## 2. 反射的两大核心：Type 和 Value

Go 中所有的变量都包含两部分信息：**类型 (Type)** 和 **值 (Value)**。`reflect` 包对应的核心也是这两个类型：`reflect.Type` 和 `reflect.Value`。

### 2.1 `reflect.TypeOf` (获取类型信息)
它接收一个 `any` 类型的参数，返回一个 `reflect.Type` 接口，里面包含了关于这个变量所有的“类型元数据”。

```go
package main

import (
    "fmt"
    "reflect"
)

func main() {
    var num float64 = 3.14
    t := reflect.TypeOf(num)
    
    fmt.Printf("类型名称(Name): %v\n", t.Name()) // 输出: float64
    fmt.Printf("类型种类(Kind): %v\n", t.Kind()) // 输出: float64
}
```

**⚠️ Name() 和 Kind() 的巨大区别：**
*   **Name (名称)**：你在代码里给它定义的类型名字。比如 `type MyInt int`，它的 Name 就是 `MyInt`。
*   **Kind (种类/分类)**：它的底层**原生归属**。对于 `type MyInt int`，它的 Kind 依然是底层的 `reflect.Int`。在反射里进行类型判断时，绝大多数情况都是用 `Kind()`！

### 2.2 `reflect.ValueOf` (获取值信息)
它返回一个 `reflect.Value` 结构体，里面包装了实际的数据，并提供了大量的方法来操纵这个数据。

```go
func main() {
    var str string = "Hello Go"
    v := reflect.ValueOf(str)
    
    fmt.Printf("反射对象的值: %v\n", v) 
    
    // 如果确认它的底层种类是 String，可以抽取出对应的原生值
    if v.Kind() == reflect.String {
        fmt.Println("抽取出原生字符串:", v.String()) 
    }
}
```

## 3. Go 反射的三大定律（Rob Pike 总结）

Go 的缔造者之一 Rob Pike 曾总结过反射的三个定律，理解它们就理解了 Go 反射的精髓。

### 定律一：反射可以将 “接口值” 转化为 “反射对象”
当你把普通的变量传给 `TypeOf` 或 `ValueOf` 时，Go 会自动把它装箱成一个空接口 `any`，然后在函数内部再拆开它，转化为反射专属的“元数据对象” (`reflect.Type` 或 `reflect.Value`)。

### 定律二：反射可以将 “反射对象” 还原回 “接口值”
通过 `reflect.Value` 提供的 `Interface()` 方法，我们可以把反射对象变回原来我们认识的普通值（但类型会变成 `any`，需要进行类型断言）。

```go
v := reflect.ValueOf(3.14) // float64 变成了 reflect.Value

// 把 reflect.Value 变回 interface{} 
i := v.Interface() 

// 原本是 float64，所以可以断言还原回来
restoreNum := i.(float64) 
fmt.Println(restoreNum)
```

### 定律三：如果要修改反射对象的值，其值必须是 “可设置的” (Settable)
这是**初学者最容易踩坑、报错 `panic`** 的地方！
想要通过反射修改一个变量的值，**必须传给反射这个变量的指针，并且还要调用 `Elem()` 解引用！**

```go
func main() {
    x := 10
    
    // ❌ 错误示范：传值
    v1 := reflect.ValueOf(x)
    // v1.SetInt(20) // 这里会直接 panic: reflect: reflect.Value.SetInt using unaddressable value
    // 为什么？因为你传的是 x 的副本，如果反射修改了这个副本，毫无意义。Go 为了防止这种无意义的操作直接禁止了。
    fmt.Println("v1 可设置吗？", v1.CanSet()) // 输出 false

    // ✅ 正确示范：传指针
    // 我们必须传 x 的地址，这样反射才知道到底去内存哪里修改
    v2 := reflect.ValueOf(&x)
    
    // 注意：v2 现在代表的是一个指针。必须要用 Elem() 拿到它指向的具体元素，才能做修改
    v_elem := v2.Elem()
    fmt.Println("v_elem 可设置吗？", v_elem.CanSet()) // 输出 true
    
    v_elem.SetInt(99) 
    fmt.Println("修改后的 x =", x) // 输出 99
}
```

## 4. 反射的最强大应用：解析结构体与 Tag (标签)

你在写 JSON 或数据库 ORM 时，经常会在结构体后面写一串反引号包起来的字符串（如 ``json:"name"` `）。这就是 Tag。只有反射才能读取它。

```go
package main

import (
    "fmt"
    "reflect"
)

type User struct {
    ID   int    `json:"id" db:"user_id"`
    Name string `json:"name" db:"user_name"`
    Age  int    `json:"age" db:"user_age"`
}

func parseStruct(obj any) {
    t := reflect.TypeOf(obj)
    v := reflect.ValueOf(obj)

    // 必须确保传入的是一个结构体（或者指向结构体的指针）
    // 如果是指针，需要解引用： t = t.Elem() 
    if t.Kind() != reflect.Struct {
        fmt.Println("传入的不是结构体")
        return
    }

    // NumField() 返回这个结构体有多少个字段
    for i := 0; i < t.NumField(); i++ {
        // t.Field(i) 拿到关于第 i 个字段的元数据（名字、类型、Tag等）
        fieldMeta := t.Field(i)
        // v.Field(i) 拿到第 i 个字段储存在里面的切实数据信息
        fieldVal := v.Field(i)

        // 读取特殊的 Tag
        jsonTag := fieldMeta.Tag.Get("json")
        dbTag := fieldMeta.Tag.Get("db")

        fmt.Printf("字段名: %s | 类型: %s | 值: %v | JSON标签: %s | DB标签: %s\n", 
            fieldMeta.Name, fieldMeta.Type, fieldVal.Interface(), jsonTag, dbTag)
    }
}

func main() {
    u := User{ID: 101, Name: "Alice", Age: 18}
    parseStruct(u)
}
```

**深入理解：** 上面的代码就是 `json.Marshal(u)` 在底层默默为你所做的事情。它用反射扫出所有的字段和值，然后根据 Tag 里的名字，拼凑出一个 JSON 格式的字符串。

## 5. 动态调用方法

除了读写字段，反射还可以寻找对象上绑定的方法，并传递动态参数触发它。

```go
type Calculator struct{}

func (c Calculator) Add(a, b int) int {
    return a + b
}

func main() {
    calc := Calculator{}
    v := reflect.ValueOf(calc)

    // 1. 通过函数名查找绑定的 Method
    method := v.MethodByName("Add")
    
    // 2. 准备参数。参数必须严格包装为 []reflect.Value 的切片
    args := []reflect.Value{
        reflect.ValueOf(10),
        reflect.ValueOf(25),
    }

    // 3. 调用方法 Call，返回值也是一个 []reflect.Value
    results := method.Call(args)
    
    // 4. 解析结果
    fmt.Printf("调用结果: %v\n", results[0].Int()) // 35
}
```

## 6. 反射的性能问题与最佳实践

反射虽然仿佛拥有黑魔法般的威力，但也伴随着巨大的代价：

1.  **慢**：反射是在运行时动态分配内存、动态去查表解析类型元数据的。通过反射读取字段、调用方法，**通常比直接写的原生代码慢上一两个数量级**（10x - 100x）。
2.  **不安全（失去静态类型检查）**：在写原生代码时，参数类型传错了，IDE 和编译器马上会爆红阻止你。但在反射里，如果你 `SetInt` 时实际上塞给它一个 `string` 或者对一个空接口调用了不存在的方法，代码能通过编译，但**一遇到这行立马运行期 Panic 宕机**。代码极其脆弱。

**实战建议：**
*   **非必要不使用**。普通的业务逻辑（增删改查、数据处理）应当使用显式的结构体和明确的类型。
*   通常只有在编写**“高度通用底层的框架或类库”**（如你试图写一个 JSON 解析器、一个 ORM 框架、一个万能的 Mock 工具）时，不知道调用者会传给你什么样的结构体，这时候才应该祭出反射这把大杀器。
